Menurut saya, naming convetion itu seperti kesepakatan cara memberi nama variabel, method, dan class agar kode terlihat rapi dan bisa dipahami oleh orang lain, termasuk diri sendiri kalau membaca ulang program di kemudian hari. Aturan naming ini juga membantu supaya kode lebih mudah dipahami saat program sudah semakin panjang. Seperti contoh pada penamaan class biasanya diawali dengan huruf kapital, seperti LatihanProduk, LatihanKalkulator, LatihanCounter, dll. Kalau penaaman pada variabel itu kata pertama diawali dengan huruf kecil dan awal kata berikutnya baru menggunakan kapital, misalnya nilaiAwal, jumlahStok, hargaSatuan, dll. Ada juga penamaan konstanta, nah ini digunakan untuk nilai yang tetap/konstan, untuk penulisannya harus ditulis uppercase (kapital) semua. Terakhir ada penamaan boolean, nah boolean ini harus menggunakan prefix yang menunjukkan ya/tidak, contoh isAvailable, isActive, canEdit, dll.

Untuk mengapa penamaan variabel dan class itu penting karena nama tersebut mewakili isi data dan fungsi program. Jadi, kalau penamaannya jelas, kita bisa langsung paham tujuan variabel atau class tanpa harus membaca seluruh kode. Misal, kalau variabel diberi nama jumlahStok atau totalHarga, kita bisa langsung tahu itu untuk apa, dibanding dengan hanya memakai nama variabel huruf a atau b saja. Begitu juga dengan class, kalau namanya sesuai dengan fungsi program, program jadi lebih terstruktur dan tidak  membingungkan saat dikembangkan lagi.

Untuk operator aritmatika di Java, digunakan untuk melakukan perhitungan angka di dalam program. Operator yang dipelajari meliputi penjumlahan, pengurangan, perkalian, pembagian, dan modulus (sisa bagi). Saya memahaminya melalui beberapa latihan yang telah saya kerjakan seperti pada latihan 2 dan 3 pada pertemuan 4. Dari latihan tersebut, saya melihat hasil operasi bisa berbeda tergantung tipe data yang digunakan. Misalnya, pembagian dengan tipe data integer hanya menghasilkan bilangan bulat, sehingga hasilnya tidak menampilkan angka desimal, contoh pada LatihanKalkulator pembagian integer dan pembagian desimal memiliki hasil yang berbeda, untuk pembagian int 25/4 itu = 6, sedangkan pada pembagian desimal itu 25/4 = 6.25. Dari sini saya menyadari bahwa tipe data sangat berpengaruh pada hasil perhitungan. 
Selain itu pada latihan counter saya mempelajari operator increment(++) dan decrement(--), serta operator penugasan seperti penambahan dan pengurangan (+=, -=, *=, /=, %=) langsung pada variabel. Operator-operetor ini membantu mempersingkat penulisan kode karena tidak perlu menuliskan operasi secara lengkap berulang kalu. Dengan menggunakan operator tersebut, perubahan nilai variabel dapat dilakukan secara lebih praktis dan mudah dipahami.

Dari latihan yang dikerjakan, saya memahami bahwa konsep variabel, konstanta, dan operator aritmatika dapat diterapkan dalam program sederhana. Pada latihan kalkulator, saya menerapkan operator aritmatika untuk menghitung penjumlahan, pengurangan, perkalian, pembagian, dan sisa bagi dari dua angka. Pada latihan counter, saya menerapkan operator increment, decrement, dan operator penugasan untuk melihat perubahan nilai variabel secara bertahap. Selain itu, pada latihan produk, saya menerapkan variabel dan konstanta untuk merepresentasikan data produk, menghitung total nilai stok, serta menghitung pajak. Dari latihan-latihan tersebut, saya memahami bagaimana konsep dasar Java bisa digunakan untuk mengolah data dan melakukan perhitungan sederhana.
